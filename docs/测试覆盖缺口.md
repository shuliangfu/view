# @dreamer/view 测试覆盖缺口清单

本文档对照 `src/` 与
`tests/`，**列举当前测试体系的薄弱点与缺失项**，便于排期补测。\
**说明**：通过不等于「已覆盖所有分支」；下列「无单测」指**没有以该模块为主角的单元/集成用例**，可能仍被间接执行。

**近期已补（2026-03-21）**：高优先级项已增加 `tests/unit/` 下对应文件（见
`docs/zh-CN/TEST_REPORT.md` 文件表）；`package.json` 的 `exports` 已补
`./ssr`，与 `deno.json` 一致，便于 Bun 解析 `@dreamer/view/ssr`。

## 五、跨运行时与环境

| 项           | 说明                                                                                                                  |
| ------------ | --------------------------------------------------------------------------------------------------------------------- |
| Deno vs Bun  | `TEST_REPORT` 中两者用例数不同（统计方式差异）；**同一关键路径在 Bun 下重复跑**可写进 CI 说明，避免「只在 Deno 绿」。 |
| 无 DOM       | Bun 未 preload `dom-setup` 时部分用例失败——**文档已说明**，缺「显式跳过或统一 preload」的测试脚手架用例。             |
| 浏览器仅 E2E | **WebSocket、剪贴板、全屏 API** 等若未来进框架，需单独浏览器策略。                                                    |

## 六、非功能项（建议后续）

| 项         | 说明                                                                                                |
| ---------- | --------------------------------------------------------------------------------------------------- |
| 代码覆盖率 | 仓库未随 CI 发布 **line/branch coverage** 门槛，无法量化「全面」。建议 `deno coverage` + 最低阈值。 |
| 性能回归   | 无基准测试；`insertReactive` 热路径可择机加 **bench**（非必须）。                                   |

---

## 七、补测优先级（建议）

1. **高（本轮已覆盖）**：`spread-intrinsic`、`insert-replacing`、`route-page`
   运行时、`compiled` 契约、**active-document / escape /
   vnode-insert-bridge**、**子路径 ssr 烟测**。
2. **中**：`src/cmd/*` 内可单测的纯函数、`router-mount` 进一步扩展（多
   Outlet、嵌套）、`hydrate` 错位场景。
3. **低**：logger 格式细节、`mod-*` 再扩、CLI 参数解析小单测。
