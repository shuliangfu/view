# 编译器优化与框架轻量化建议

在「全量 .tsx 编译 + insert 仅 (parent)=>void / 基础类型」的路线 C
下，以下方向可进一步把逻辑前移到编译期、收缩运行时，让框架更轻量。

---

  按需导入。

## 一、编译器补齐能力（减少运行时与缺失能力）

### 1.1 事件绑定（onClick / on*）

**现状**：`buildAttributeStatements` 对 `on*`
属性直接跳过（`isEvent`），编译产物里没有事件绑定。

**建议**：在编译器里对 `onClick`、`onChange` 等生成
`el.addEventListener("click", handler)`（或先存到
`el.__view$on = { click: handler }` 再在下一帧/入文档后统一绑定，与旧 applyProps
行为一致）。

- 这样**不依赖**已删除的
  dom/props，事件逻辑全部在编译产物里，运行时无需再提供「应用 props」的通用层。

### 1.2 ref

**现状**：未处理 `ref`。

**建议**：在 `buildAttributeStatements` 后追加一段：若存在 `ref`，生成\
`if (ref != null) typeof ref === "function" ? ref(el) : (ref.current = el);`\
这样 ref 完全由编译期完成，无需运行时 API。

### 1.3 style 对象

**现状**：仅对字面量/表达式做 `setAttribute(name, val)`，未处理
`style={{ color: "red" }}`。

**建议**：对 `style` 为对象表达式时，生成对 `el.style.xxx` 的赋值或
`Object.assign(el.style, style)`，避免运行时再解析 style 对象。

### 1.4 布尔 / 通用 attribute

**现状**：主要处理 className/class 和一般 setAttribute，未专门处理
`disabled`、`checked`、`hidden` 等。

**建议**：对已知布尔属性在编译期生成 `el.disabled = !!val` 等，或统一走
setAttribute；减少运行时分支。

## 二、编译器承担更多 → 运行时更薄

### 2.1 组件返回值「挂载函数」在编译期展开

**现状**：运行时 `insert` 对 getter 有分支：若 `getter()` 返回
`(parent) => void`，则创建 wrapper div、挂到 parent、再执行该函数。

**建议**：组件调用处由编译器生成「先创建 wrapper（或直接挂到 parent）、再调用
`Comp(props)(parentOrWrapper)`」，不再把「返回 (parent)=>void」的 getter
交给运行时。

- 运行时 `insert` 只需处理：**静态值**（string/number/Node/null/undefined）与
  **getter → 上述类型**，可删除「getter 返回 (parent)=>void」和 wrapper div
  逻辑，运行时更简单、分支更少。

### 2.2 静态插入不经过 insert

**现状**：静态文本也走 `insert(parent, "hello")`，运行时再 `toNodeForInsert` +
appendChild。

**建议**：编译器对**纯静态**子节点直接生成\
`parent.appendChild(document.createTextNode("hello"));`\
不调用 `insert`。

- 这样静态路径不依赖运行时的 `toNodeForInsert`，且有利于
  tree-shaking：若某组件无动态子节点，可只依赖 `appendChild` 而不必拉取 `insert`
  的 getter/effect 逻辑（若再配合 2.3 拆分 API 则更明显）。

### 2.3 拆成三个运行时原语（可选）

**现状**：一个 `insert(parent, value)`
同时处理：静态值、getter、`(parent)=>void`。

**建议**：提供三个小 API，由编译器按需选用：

- `appendChild(parent, node)` 或 `insertStatic(parent, value)`：仅静态，无
  effect。
- `insertReactive(parent, getter)`：内部 createEffect + toNode + replace，仅在有
  `{ expr }` 的插入点需要。
- `mount(parent, fn)`：`fn(parent)`，仅组件/portal 需要。

编译器根据「是否动态」「是否组件」选择调用其一，主入口 `insert`
可保留为三者的小包装。

- **好处**：无动态子节点的组件不会拉取 createEffect/toNode/replace
  逻辑，包更小；运行时实现更薄、更易内联。

## 三、可移除或迁出主路径的代码（进一步轻量化）

### 3.1 normalizeChildren / dom/element / 部分 dom/shared

**现状**：`normalizeChildren` 仅由 **`runtime/vnode-mount.ts`**（及
`dom/element` 自身）使用，**`runtime.ts` 主文件不再直接 import**；VNode
子树内嵌套 `insertReactive` 通过 **`vnode-insert-bridge`**
在模块末尾注册，避免循环依赖。

**说明**：

- 独立的 `@dreamer/view/dom-legacy` 已移除；需要 `normalizeChildren` 时请从
  `@dreamer/view/dom` 或 `dom/element.ts` 导入。
- **`@dreamer/view/runtime`**（`runtime/mod.ts` → `insert.ts`）仍**不含** VNode
  展开与 `normalizeChildren`；主包 **`@dreamer/view`** 的 `insertReactive` 含
  VNode 分支时会加载上述子模块。
- 若未来 Suspense 等也改为纯 `(parent)=>void` 且不再产出 VNode，可再评估删除
  `vnode-mount` 与收缩 `normalizeChildren`。

### 3.2 jsx-runtime（jsx/jsxs/Fragment）

**现状**：全量编译后，.tsx 被转成
`(parent) => { … insert(parent, …) }`，不再调用
jsx/jsxs；但未编译代码（测试、手写 JSX）仍依赖 jsx-runtime。

**建议**：

- 主包或「编译专用」入口可不导出或按需导出 jsx-runtime；打「仅编译产物」的
  bundle 时可不包含 jsx-runtime，进一步减体积。
- 测试或兼容层单独从 `@dreamer/view/jsx-runtime` 拉取即可。

### 3.3 directive（v-if / v-show / v-for）

**现状（已部分前移到编译器）**：内置元素上 `v-if` / `v-show` / `vFor`（或
`v-for`）已由编译器生成条件块、`style.display` / `createEffect`、以及
`insertReactive(..., () => list.map(...))` 等；未写指令属性的场景仍可用
`{ cond() && <A /> }`、`list().map(...)`。

**建议（可选深化）**：

- **方案 A**：在编译器里实现等价逻辑——如 `v-if` 编译为条件分支或三元、`v-show`
  编译为 `el.style.display = show() ? "" : "none"`、`v-for` 编译为 map + key
  的插入循环。之后可将 directive 中与 v-if/v-show/v-for
  相关的实现标为可选或兼容层，主路径不依赖。
- **方案 B**：不实现上述指令，保持「条件/列表用 JS 表达式」，则 directive
  仅作为可选扩展或兼容保留，不参与主路径，主包也可不打包其实现。

### 3.4 insertReplacing

**现状**：`insertReplacing` 已迁至 **`runtime/insert-replacing.ts`**，子路径
`@dreamer/view/runtime/insert-replacing`；主 `runtime/mod` 与 `insert.ts`
不导出；编译器未使用。

**建议**：占位替换场景按需从子路径导入即可。

## 四、建议优先级（轻量化收益 vs 实现成本）

| 优先级 | 项                                                            | 收益 | 说明                                          |
| ------ | ------------------------------------------------------------- | ---- | --------------------------------------------- |
| 高     | 1.1 事件绑定                                                  | 大   | 补齐能力且不再依赖已删的 dom/props            |
| 高     | 2.1 组件返回值在编译期展开                                    | 中   | 运行时去掉 wrapper div 与 getter 返回函数分支 |
| 中     | 1.2 ref、1.3 style                                            | 中   | 属性完整、行为可预测                          |
| 中     | 2.2 静态插入不经过 insert                                     | 中   | 静态路径更轻、利于 tree-shake                 |
| 中     | 3.1 normalizeChildren 迁出主路径                              | 中   | 主包更小、边界清晰                            |
| 低     | 2.3 三原语拆分                                                | 中   | 需改编译器多处调用，收益主要在包体积          |
| 低     | 3.2 jsx-runtime 按需、3.3 directive 可选、3.4 insertReplacing | 小   | 打包/入口配置与可选加载即可                   |

## 五、小结

- **编译器多做一点**：事件、ref、style、布尔属性、组件挂载形态、静态插入形态，能前移的都前移，运行时只保留「最少必要」的
  insert/effect/append/mount。
- **运行时只做必要分支**：去掉「getter 返回 (parent)=>void」和 wrapper
  div，必要时拆成 appendStatic + insertReactive + mount 三个薄 API。
- **主路径不背未用代码**：normalizeChildren/VNode
  兼容、jsx-runtime、directive、insertReplacing
  等按「编译专用」与「兼容/可选」拆开，主 bundle 只带编译产物真正用到的部分。

这样框架会朝「能放编译期的都放编译期、运行时尽量薄、主包尽量小」的方向更轻量化。

## 六、打包与按需说明（3.2 / 3.3 / 3.4）

- **3.2 jsx-runtime**：全量编译后业务 .tsx 不再调用
  jsx/jsxs。打「仅编译产物」bundle 时，可从入口中排除
  `@dreamer/view/jsx-runtime`，仅测试或未编译代码按需从
  `@dreamer/view/jsx-runtime` 拉取。
- **3.3 directive**：内置元素上的 `v-if` / `v-else-if` / `v-else`
  链、`v-show`、`vFor`、`v-once`、`v-cloak` 已在编译器中实现（见
  `jsx-compiler/transform.ts`）；自定义 `registerDirective` 与 v-insert 仍用
  directive 或运行时 API。
- **3.4 insertReplacing**：已由主 `@dreamer/view/runtime` **迁出**，请从
  `@dreamer/view/runtime/insert-replacing` 按需导入；编译器未使用。

## 七、内置指令：编译器（内置标签）与 `@dreamer/view/directive`

| 能力                              | JSX 编译器 `compileSource`（内置 HTML 标签）                                                         | `@dreamer/view/directive`（运行时 VNode / 手写 props） |
| --------------------------------- | ---------------------------------------------------------------------------------------------------- | ------------------------------------------------------ |
| v-if / v-if                       | ✅ 单节点条件块；或与兄弟 **vElseIf / vElse** 组成链（链内须均为**内置标签**）                       | ✅                                                     |
| v-else / v-else-if                | ✅ 仅作 **v-if 起头的兄弟链**                                                                        | ✅                                                     |
| v-for / vFor                      | ✅ 限定：`vFor={列表}` + **唯一**子节点为 `(item,i)=>单棵JSX`；`() => arr` 会编译为立即调用          | ✅ 更通用形态                                          |
| v-show / v-show                   | ✅ style.display / createEffect；**v-once** 子树内对 effect 体包 `untrack`                           | ✅                                                     |
| v-once / v-once                   | ✅ 子树内 `insertReactive` 等 getter 包 **`untrack`**（需从 `@dreamer/view/runtime` 引入 `untrack`） | ✅                                                     |
| v-cloak / v-cloak                 | ✅ `setAttribute("data-view-cloak","")`，与根 `createRoot` 后 `removeCloak` 一致                     | ✅                                                     |
| v-insert / 占位替换               | ❌ 请 `insert(el, getter)` 或 `@dreamer/view/runtime/insert-replacing`                               | ✅                                                     |
| 自定义 `v-*`（registerDirective） | ❌ 注册表在运行时，无法在编译期展开                                                                  | ✅                                                     |

**结论**：**内置结构性指令**（if/else 链、for、show、once、cloak）在全编译 +
内置标签路径下已由编译器覆盖；**自定义指令、v-insert** 仍依赖
**`@dreamer/view/directive`** 或手写 API。
